
# Slydo：企业级智能 PPT 知识库系统设计说明书 (V3.0)

## 1. 项目愿景与目标

**Slydo** 是一款深度集成于 PowerPoint 的 AI 助手，旨在通过 **Agentic Graph-RAG** 技术，将企业海量（万页级）PPT 资产转化为可预测、可复用的智能逻辑网络。它不仅仅是“搜素材”，更是通过理解全篇大纲和页面语义，引导用户完成高质量的专业演示文稿创作。

---

## 2. 全局技术架构

系统采用“插件端沉浸交互 + 云端智能推理 + 双引擎存储”的构架。

### 2.1 存储双引擎

1. **向量数据库 (Vector DB - Milvus)：** 存储页面语义摘要（Embedding），负责处理“含义相似性”。
2. **图数据库 (Graph DB - Neo4j)：** 存储文档拓扑关系（如 `Next_Slide_Of`、`Belongs_To_Category`），负责处理“逻辑连贯性”。

### 2.2 逻辑中枢：Agentic 推理层

利用 AI Agent 框架（如 LangGraph）调度检索任务，实现：

* **语义扩展：** 将短标题扩展为深度检索指令。
* **路径预测：** 基于图谱预测下一页的最佳逻辑。
* **多维重排：** 结合语义、逻辑和质量分（QS）进行结果优选。

---

## 3. 万页级数据入库与预处理 (ETL)

针对企业存量 PPT 资产，进行深度清洗与知识萃取：

* **三维解析：** 提取 **结构化 XML**（标题/正文）、**备注页**（逻辑重心）及 **视觉特征**（利用多模态模型生成布局描述）。
* **逻辑萃取：** LLM 自动识别幻灯片在原文档中的角色（如：封面、目录、转场、结论），并在图数据库中建立逻辑链条。
* **去重聚类：** 识别万页数据中的重复模板，合并冗余项，建立“标准母版”关联。

---

## 4. 核心交互逻辑 (The Trigger Protocol)

系统采用“精准触发、低干扰”的交互设计，分为三个核心场景：

### 场景 A：自动推荐（标题驱动模式）

* **触发条件：** 用户切换至新页面；**检测到标题占位符有内容** 且 **正文占位符为空**。
* **逻辑：** 插件静默抓取标题 -> 后端进行短文本向量检索 -> 侧边栏自动展示 3-5 张高匹配页面。
* **目标：** 解决“有了标题不知道怎么画内容”的问题。

### 场景 B：手动/大纲推荐（逻辑预测模式）

* **触发条件：** 页面完全空白（无标题、无正文）；用户在插件端手动点击“**推荐页面**”按钮。
* **逻辑：**
    1. Agent 扫描当前 PPT 的全局大纲（所有已完成页面的标题序列）。
    2. 结合前 2-3 页的正文摘要，利用 LLM 推理下一页的 3 个潜在逻辑走向。
    3. 从图数据库中检索出符合这些逻辑走向的“标杆页面”。
* **目标：** 解决“不知道下一步写什么”的逻辑断层问题。

### 场景 C：手动搜索模式

* **触发条件：** 用户在搜索框输入自定义关键词。
* **逻辑：** 标准向量检索 + 关键词检索（BM25），支持按部门、时间、风格进行筛选。

---

## 5. “金牌页面”质量管理体系

为确保万页库中的优质资产优先浮现，系统引入 **Quality Score (QS)** 机制。

### 5.1 评分权重维度

* **官方认证 (Weight 0.5)：** 管理员手动标记的“标准模板”。
* **复用频率 (Weight 0.3)：** 页面被其他员工导入、下载的实时统计。
* **源文档等级 (Weight 0.2)：** 存储在核心方案库、最终版文件夹中的内容。

### 5.2 加权重排序公式

检索结果不仅依赖语义相似度，还需通过质量分重排：
$$Final\_Score = (\alpha \cdot Semantic\_Similarity) + (\beta \cdot Quality\_Score)$$
其中，$\alpha$ 和 $\beta$ 为可动态调整的系数，确保在相关性的基础上优先推荐“金牌资产”。

---

## 6. 插件端功能实现 (VSTO Add-in)

* **UI/UX：** 侧边栏任务面板（Task Pane）采用流式布局展示推荐页面的高清缩略图。
* **一键导入：** 点击缩略图即调用 `Slide.InsertFromFile`，并提供“保持源格式”或“跟随当前模板”两个选项。
* **智能更新：** 若用户当前使用的页面在库中有更高级的“金牌版本”，插件右下角弹出轻量级更新提示。
* **金牌标识：** 缩略图右上角通过星标展示 Quality 认证。

## 7. 技术栈选型参考

| 模块 | 技术栈 |
| :--- | :--- |
| **客户端** | C# (VSTO / .NET 4.8+) |
| **后端 API** | Python 3.11 (FastAPI) |
| **向量库** | Milvus (集群版，支持 10k+ 维向量) |
| **图数据库** | Neo4j (用于存储逻辑路径) |
| **Embedding** | BGE-M3 (支持多粒度、长短文本检索) |
| **大模型** | Gemini 1.5 Pro / GPT-4o (逻辑推理与 Agent 调度) |
| **Rerank** | BGE-Reranker-v2 (用于最后 50 条结果的精排) |

---

## 8. 实施路径 (Roadmap)

1. **数据清洗期 (W1-W3)：** 处理万页 PPT，建立向量库与初步图数据库。
2. **插件 MVP (W4-W6)：** 实现 C# 插件与场景 A 的自动触发逻辑。
3. **智能升级期 (W7-W10)：** 引入 Agentic 框架，实现场景 B 的大纲推理与金牌页面评分系统。
4. **企业集成期 (W11+)：** SSO 登录集成、权限隔离、离线缓存优化。

---

**Slydo** 的设计核心在于将企业沉淀的“静止资产”转化为“活的逻辑”。通过精准的触发时机和金牌质量体系，它将极大地缩短员工从构思到成品的时间。
