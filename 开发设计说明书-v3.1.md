## 1. 设计哲学

### 1.1 核心理念

**"含义优先"**：Slydo 的核心是对 PPT 页面语义含义的提取、存储与匹配，而非简单的文本检索。

系统理解每一页 PPT 在文档中的**角色**（封面、目录、转场、核心论点、论据、结论）和**语义含义**（这一页在"说什么"），新制作 PPT 时根据当前页面的语境推荐语义匹配的历史页面。

### 1.2 为什么不需要图数据库

PPT 页面之间的"逻辑关系"是**上下文敏感的**，不是固定不变的节点-边关系：

| 所谓"图关系" | 实际形态 | 实现方式 |
|:---|:---|:---|
| `Next_Slide_Of` | 同一文档内的前后页 | PostgreSQL 存 `deck_id + slide_index` |
| `Belongs_To_Category` | 标签分类 | PostgreSQL 多对多关联表 |
| "逻辑走向" | 用户在创作时的动态决策 | LLM 在查询时根据**语义+上下文**推理 |

**关键判断**：预存固定的图关系会限制推荐灵活性 —— 同一张"市场分析"页，在 A 方案中是背景铺垫，在 B 方案中是竞争对手对比。LLM 在查询时动态推理比预先建图更灵活。

### 1.3 为什么用 LLM Wiki

LLM Wiki（Karpathy 提出的模式）是一种持久化的 Markdown 知识库，在这里替代两个角色：

1. **结构化元数据存储**：每张 PPT 页面的含义摘要、文档上下文、标签、质量评分
2. **逻辑推理层**：LLM 在查询时根据 Wiki 上下文动态推理推荐结果，替代预定义的"图路径"

### 2.1 存储双引擎（轻量化版）

| 引擎 | 用途 | 选型理由 |
|:---|:---|:---|
| **Qdrant** | 页面语义向量检索 | 无 Docker 依赖可用本地模式（RocksDB），1024 维向量，十万级规模无压力 |
| **PostgreSQL** | 结构化数据 + FTS 全文搜索 | 运维极简，FTS 支持中文分词（zhparser），一张表搞定所有关系 |

> **为什么不用 Milvus？** 初创项目不需要分布式集群，Qdrant 本地模式零运维负担。
> **为什么不用 Neo4j？** 需要存的"关系"用 PostgreSQL 外键 + ORDER BY 即可解决。

### 2.2 逻辑中枢：LLM Wiki + LLM 推理

由 LLM 在**查询时动态编排**推荐流程：

```
用户触发推荐
    │
    ▼
① Qdrant 语义检索 → 召回 Top-K 语义相近页面
    │
    ▼
② PostgreSQL FTS → 关键词补充召回（含 visual_desc）
    │
    ▼
③ LLM Wiki 提供候选页面的上下文（含义摘要、文档背景、前驱后继、标签）
    → 作为 LLM 重排步骤的上下文输入
    │
    ▼
④ LLM 理解当前上下文（前几页标题+摘要）
    → 判断用户在"讲什么"
    → 结合①+②的候选 + ③的 Wiki 上下文，推理"下一页最合适的逻辑走向"
    → 输出推荐理由
    │
    ▼
⑤ 返回 Top-N 推荐页面 + LLM 注释
```

## 3. 数据模型

### 3.1 PostgreSQL 核心表

```sql
-- ============================================
-- 文档（PPT 文件）
-- ============================================
CREATE TABLE decks (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title           TEXT NOT NULL,                -- 文档标题
    file_path       TEXT,                         -- 源文件路径（可选）
    department      TEXT,                         -- 所属部门
    category        TEXT,                         -- 分类标签
    slide_count     INTEGER NOT NULL DEFAULT 0,
    checksum        TEXT,                         -- 文件 MD5，用于去重
    version         INTEGER DEFAULT 1,
    is_official     BOOLEAN DEFAULT FALSE,        -- 是否官方认证（QS 使用）
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- 幻灯片页面（核心表）
-- ============================================
CREATE TABLE slides (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    deck_id         UUID NOT NULL REFERENCES decks(id) ON DELETE CASCADE,
    slide_index     INTEGER NOT NULL,             -- 在文档中的页码（从 1 开始）
    title           TEXT,                         -- 原始标题文本
    body_text       TEXT,                         -- 原始正文文本（用于 FTS 索引）
    notes_text      TEXT,                         -- 备注页文本（逻辑重心）
    
    -- 语义含义（ETL 过程中由 LLM 提取）
    semantic_role   TEXT,                         -- 角色：cover/toc/transition/argument/evidence/conclusion/appendix
    semantic_summary TEXT,                        -- 含义摘要（1-3 句话）
    semantic_tags   TEXT[],                       -- 语义标签数组
    
    -- 视觉特征
    layout_type     TEXT,                         -- 布局类型（标题+正文/标题+图表/全图/左文右图/...）
    visual_desc     TEXT,                         -- 视觉模型生成的布局描述（配色、图表类型、排版）
    thumbnail_path  TEXT,                         -- 缩略图路径（wiki/slydo/thumbnails/...）
    
    -- 质量评分
    usage_count     INTEGER DEFAULT 0,            -- 被导入次数
    quality_score   REAL DEFAULT 0.0,             -- 综合质量分
    
    -- 向量（存储于 Qdrant）
    qdrant_point_id TEXT,                         -- Qdrant 中的 point ID
    
    -- 向量嵌入的文本来源：Phase 4 使用 semantic_summary + semantic_role + visual_desc
    -- 拼接的字符串做嵌入，而非原始 body_text。这样纯图片页(无文本)也能通过
    -- visual_desc + semantic_summary 做语义匹配。
    
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_slides_deck_id ON slides(deck_id);
CREATE INDEX idx_slides_semantic_role ON slides(semantic_role);
CREATE INDEX idx_slides_tags ON slides USING GIN(semantic_tags);

-- 中文全文搜索索引（需安装 zhparser）
-- CREATE TEXT SEARCH CONFIGURATION chinese_zh (PARSER = zhparser);
-- CREATE INDEX idx_slides_fts ON slides USING GIN(to_tsvector('chinese_zh', 
--     COALESCE(title,'') || ' ' || COALESCE(body_text,'') || ' ' 
--     || COALESCE(semantic_summary,'') || ' ' || COALESCE(visual_desc,'')
-- ));

-- ============================================
-- 分类标签
-- ============================================
CREATE TABLE tags (
    id              SERIAL PRIMARY KEY,
    name            TEXT UNIQUE NOT NULL,
    category        TEXT                          -- 标签分类（如：行业、场景、格式）
);

-- 幻灯片-标签关联
CREATE TABLE slide_tags (
    slide_id        UUID REFERENCES slides(id) ON DELETE CASCADE,
    tag_id          INTEGER REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (slide_id, tag_id)
);

-- ============================================
-- 使用日志（用于 QS 评分）
-- ============================================
CREATE TABLE usage_log (
    id              BIGSERIAL PRIMARY KEY,
    slide_id        UUID REFERENCES slides(id) ON DELETE CASCADE,
    user_id         TEXT,
    action          TEXT NOT NULL,                -- 'view' / 'import'
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_usage_log_slide ON usage_log(slide_id);
```

### 4.1 入库流水线

```

  ▼
┌──────────────────────────────────────────────────────┐
│ Phase 1: 文档解析 + 视觉渲染                          │
│  · python-pptx 提取 XML 文本（标题/正文/备注）        │
│  · 将每页渲染为 PNG 图片（生成可视化的缩略图）         │
│  · 提取布局信息（占位符布局类型）                      │
│  · 计算文件 checksum（去重）                          │
│  · 补充：纯图片页/截图页 → python-pptx 可能提取       │
│    不到文本，但 Phase 2 的视觉模型仍可分析含义         │
└─────────────────────┬──────────────────────────────┘
                      │  文本（标题/正文/备注）
                      │  + 页面图片（PNG）
                      ▼
┌──────────────────────────────────────────────────────┐
│ Phase 2: 多模态 LLM 含义提取                          │

│  · **注意：纯图片页（无文本）也能通过视觉模型分析含义** │
│  · 批处理调用，组合同一批 PPT 的页面减少 API 往返      │
└─────────────────────┬──────────────────────────────┘
                      ▼
┌──────────────────────────────────────────────────────┐

┌──────────────────────────────────────────────────────┐
```

> **关键设计决策**：向量嵌入基于"LLM 输出的含义摘要"而非"原始文本"。这意味着：
> - 纯图片页（无文本）→ 视觉模型输出含义摘要 → 同样可做语义匹配
> - 文本页 → LLM 提炼后的摘要去除了噪声，语义更精确
> - 搜索时也是语义对语义，而非文本片段对文本片段

### 4.2 多模态 LLM 含义提取 Prompt

Phase 2 接收的输入包括：**页面截图（PNG）+ 提取到的文本**。
如果是纯图片页，文本可能为空，但视觉模型仍可分析。

#### 输入结构

```json
{
  "slide_index": 5,
  "text_content": {
    "title": "银行业务数字化转型面临的挑战",
    "body": "获客成本高：线上流量成本增长300%...",
    "notes": "强调数据支撑，引用2024年行业报告数据"
  },
  "image_available": true
}
```

#### LLM Prompt（视觉模型）

```text
你正在分析一个 PPT 页面的截图及其提取的文本，请给出：

1. **页面角色**：是以下哪种？
   - cover(封面) / toc(目录) / transition(转场页) / argument(核心论点)
   - evidence(论据/数据支撑) / conclusion(结论页) / appendix(附录)

2. **含义摘要**：用 1-3 句话概括这一页在"说什么"（不是原文摘录，而是理解后的语义含义）

3. **视觉描述**：简述页面视觉结构（如：左文右图结构，饼图展示市场份额占比，深蓝商务风配色）

4. **语义标签**：3-8 个标签，覆盖：行业、场景、分析维度、关键词

【页面信息】
截图通过视觉输入提供。
文本提取结果（可能为空）：
标题：{title}
正文：{body_text}
备注：{notes_text}
```

### 4.3 页面渲染为图片的技术方案

将 PPT 页面导出为 PNG 缩略图有多种方案，按推荐优先级排列：

**方案一（推荐）：python-pptx + LibreOffice 命令行渲染**
```bash
# Step 1: PPT → PDF（LibreOffice 命令行）
libreoffice --headless --convert-to pdf input.pptx --outdir /tmp/
# Step 2: PDF → PNG 每页（PyMuPDF / pdf2image）
python3 -c "
from pdf2image import convert_from_path
images = convert_from_path('/tmp/input.pdf', dpi=150)
for i, img in enumerate(images):
    img.save(f'output_page_{i+1}.png', 'PNG')
"
```

**方案二（纯 python-pptx 导出）+ LibreOffice 回退**
python-pptx 本身无法将页面渲染为图片（它只操作 XML），但可结合 COM 接口（仅 Windows）或调用 LibreOffice 完成。

**方案三（企业环境有 PowerPoint 时—Windows 端入库）**
如果在 Windows 环境执行入库，可以直接用 PowerPoint COM 对象：
```python
import win32com.client
app = win32com.client.Dispatch("PowerPoint.Application")
pres = app.Presentations.Open(pptx_path)
for i, slide in enumerate(pres.Slides):
    slide.Export(f"page_{i+1}.png", "PNG")
```

### 4.4 PPT 单页导入机制（VSTO Save + InsertFromFile）

PowerPoint 的 `Slide.InsertFromFile()` API 只能整份导入 .pptx 文件，无法直接导入单页。因此需要**后端生成临时单页 PPT**，插件再调用 `InsertFromFile`。

#### 4.4.1 流程

```
用户在插件侧边栏点击推荐页面的缩略图
    │
    ▼
① 插件 GET /api/slides/{id}/export → 后端处理
    │
    ▼
② 后端 python-pptx 生成临时单页 PPT
   · 新建空白 Presentation
   · 读取源文件的 slide_layout（保持样式）
   · 克隆该页的 XML（shapes、文本、图片、表格、图表）
   · 保存为临时文件：/tmp/slydo_export/slide_{slide_id}.pptx
    │
    ▼
③ 后端返回临时文件的下载路径（或文件流）
```

#### 4.4.2 后端导出单页 PPT 的实现（Python）

```python
from pptx import Presentation
from pptx.util import Inches, Pt
from pptx.enum.shapes import MSO_SHAPE_TYPE
from lxml import etree
import io
import os, uuid

def export_single_slide(deck_path: str, slide_index: int) -> str:
    """
    从源 PPT 中导出单页为临时 PPT 文件。
    返回临时文件路径。
    """
    # Step 1: 打开源文档
    src_prs = Presentation(deck_path)
    src_slide = src_prs.slides[slide_index - 1]  # 0-based
    
    # Step 2: 新建空白 Presentation（使用源文档的母版）
    tmp_prs = Presentation()
    # 复制源文档的 slide layout（保持源样式）
    # python-pptx 的 clone_slide_layout() 需要额外实现
    # 简化方案：使用默认 blank layout + 逐个复制形状
    
    # Step 3: 复制形状
    tmp_slide = tmp_prs.slides.add_slide(tmp_prs.slide_layouts[6])  # blank
    for shape in src_slide.shapes:
        if shape.shape_type == MSO_SHAPE_TYPE.PICTURE:
            # 复制图片
            image = shape.image
            image_bytes = image.blob
            left, top = shape.left, shape.top
            width, height = shape.width, shape.height
            tmp_slide.shapes.add_picture(
                io.BytesIO(image_bytes), left, top, width, height
            )
        elif shape.has_text_frame:
            # 复制文本框
            txBox = tmp_slide.shapes.add_textbox(
                shape.left, shape.top, shape.width, shape.height
            )
            tf = txBox.text_frame
            tf.word_wrap = shape.text_frame.word_wrap
            for para in shape.text_frame.paragraphs:
                p = tf.add_paragraph()
                p.text = para.text
                p.font.size = para.font.size
                p.font.bold = para.font.bold
                p.font.color.rgb = para.font.color.rgb
                p.alignment = para.alignment
    
    # Step 4: 保存临时文件
    tmp_dir = "/tmp/slydo_export/"
    os.makedirs(tmp_dir, exist_ok=True)
    tmp_path = os.path.join(tmp_dir, f"slide_{uuid.uuid4().hex[:8]}.pptx")
    tmp_prs.save(tmp_path)
    return tmp_path
```

> **注意：** 上述 `python-pptx` 实现对于复杂排版（表格、图表、SmartArt、组合形状等）可能有兼容性问题。对于 MVP 阶段可先支持纯文本 + 图片页面，后续逐步覆盖复杂元素。如果需要完整保真度，可考虑在 Windows 环境下使用 PowerPoint COM 对象导出单页。

### 4.5 增量更新：目录监控 + 双版本管理 + 删除清理（Phase D）

#### 4.5.1 目录监控

使用 `watchdog`（Linux）或 `FileSystemWatcher`（Windows）监控指定目录的文件变化：

| 事件 | 处理逻辑 |
|:---|:---|
| **文件新增/修改** | 计算 checksum，与数据库中已有记录比对。若 checksum 不同 → 触发增量入库流程 |
| **文件删除** | 根据 `file_path` 匹配 decks 表，级联清理关联数据 |

```python
# watcher.py - 目录监控守护进程
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler
import hashlib

class PptWatcher(FileSystemEventHandler):
    def __init__(self, api_base_url: str):
        self.api_base_url = api_base_url
        
    def on_created(self, event):
        if event.src_path.endswith(('.pptx', '.ppt')):
            self._ingest(event.src_path)
    
    def on_modified(self, event):
        if event.src_path.endswith(('.pptx', '.ppt')):
            self._re_ingest_if_changed(event.src_path)
    
    def on_deleted(self, event):
        if event.src_path.endswith(('.pptx', '.ppt')):
            self._cleanup(event.src_path)
    
    def _ingest(self, path):
        requests.post(f"{self.api_base_url}/api/ingest", json={"path": path})
    
    def _re_ingest_if_changed(self, path):
        # 计算当前 checksum
        new_checksum = hashlib.md5(open(path, 'rb').read()).hexdigest()
        # 与 DB 中该文件的 checksum 比对
        resp = requests.get(f"{self.api_base_url}/api/decks/by_path", 
                          params={"path": path})
        if resp.ok and resp.json().get("checksum") != new_checksum:
            requests.post(f"{self.api_base_url}/api/ingest", 
                        json={"path": path, "force": True})
    
    def _cleanup(self, path):
        requests.delete(f"{self.api_base_url}/api/decks/by_path",
                      params={"path": path})

# 启动监控
observer = Observer()
handler = PptWatcher("http://localhost:8000")
observer.schedule(handler, "/path/to/ppt_library", recursive=True)
observer.start()
```

#### 4.5.2 双版本管理

一份 PPT 源文件保留两个版本：

| 版本 | 说明 | 存储方式 |
|:---|:---|:---|
| **当前版 (v_current)** | 最新入库的版本 | DB 中 `decks.version` 标记为当前版本 |

```sql
CREATE TABLE deck_versions (
```

## 6. "金牌页面"质量管理体系 (QS)

### 6.1 评分维度

| 维度 | 权重 | 说明 |
|:---|:---|:---|
| **官方认证** | 0.40 | 管理员手动标记的"标准模板" |
| **复用频率** | 0.30 | 页面被导入使用次数（usage_log 统计） |
| **源文档等级** | 0.20 | 存储在核心方案库 vs. 草稿文件夹 |
| **LLM 质量评估** | 0.10 | LLM 对页面"内容完整度/视觉质量"的主观评分 |

### 6.2 排序公式（修正版）

由于 Semantic Similarity（0~1）和 Quality Score（0~1）量纲一致：

$$FinalScore = \alpha \cdot SemanticSimilarity + \beta \cdot QualityScore$$

其中 $\alpha + \beta = 1$，系统默认 $\alpha=0.7, \beta=0.3$（相关性优先，质量辅助），可由管理员根据企业偏好调节。

### 6.3 QS 更新机制

- **每日离线任务**：汇总 usage_log，更新 slides.usage_count 和 quality_score
- **管理员手动标记**：直接设置 is_official = true，QS 即时刷新
| **新入库默认值** | is_official=false，usage_count=0，quality_score=0.0 |

> **QS 计算实现参考（SQL）**：
> ```sql
> -- 每日 QS 更新（cron 每日凌晨执行）
> WITH usage_stats AS (
>   SELECT slide_id,
>          COUNT(*) FILTER (WHERE action = 'import') AS import_count,
>          COUNT(*) FILTER (WHERE action = 'view') AS view_count
>   FROM usage_log
>   WHERE created_at >= NOW() - INTERVAL '7 days'
>   GROUP BY slide_id
> )
> UPDATE slides s SET
>   usage_count = COALESCE((
>     SELECT import_count FROM usage_stats WHERE slide_id = s.id
>   ), 0),
>   quality_score = (
>     0.40 * CASE WHEN s.is_official THEN 1.0 ELSE 0.0 END +
>     0.30 * LEAST(COALESCE((SELECT import_count FROM usage_stats WHERE slide_id = s.id), 0) / 10.0, 1.0) +
>     0.20 * 0.5 +  -- 源文档等级（简化版，生产环境按目录分级）
>     0.10 * 0.5    -- LLM 质量评估（简化版）
>   )
> WHERE s.id IN (SELECT slide_id FROM usage_stats);
> ```

```python
def retry(max_retries=3, delay=2, backoff=2):
    """带指数退避的重试装饰器
    注意：生产环境建议增加熔断器（circuit breaker），连续失败 N 次后跳过该操作一段时间。
    """
    def decorator(func):
        def wrapper(*args, **kwargs):
            last_exc = None
            for attempt in range(max_retries):
                try:
                    return func(*args, **kwargs)
                except (requests.RequestException, json.JSONDecodeError, 
                        subprocess.CalledProcessError) as e:
                    last_exc = e
                    if attempt < max_retries - 1:
                        sleep_time = delay * (backoff ** attempt)
                        log.warning(f"重试 {attempt+1}/{max_retries}: {e}, {sleep_time}s后重试")
                        time.sleep(sleep_time)
                    else:
                        log.error(f"重试耗尽: {e}")
                        raise
            raise last_exc
        return wrapper
    return decorator


class TokenCounter:
    """简单的 Token 估算（非精确计数，仅用于成本监控）"""
    total_input = 0
    total_output = 0
    
    @classmethod
    def estimate(cls, text: str) -> int:
        """粗略估算：中文约1.5字符/token，英文约4字符/token"""
        chinese = sum(1 for c in text if '\u4e00' <= c <= '\u9fff')
        english = len(text) - chinese
        return int(chinese * 1.5 + english / 4)
    
    @classmethod
    def add_usage(cls, model: str, input_tokens: int, output_tokens: int):
        cls.total_input += input_tokens
        cls.total_output += output_tokens
    
    @classmethod
    def report(cls) -> str:
        total_cost = 0.0
        details = []
        for model, rates in TOKEN_COST.items():
            # 简化：按总输入/输出均分到各模型
            cost = (cls.total_input / 1_000_000 * rates["input"] + 
                    cls.total_output / 1_000_000 * rates["output"])
            total_cost += cost
            details.append(f"  {model}: 输入={cls.total_input:,}t, 输出={cls.total_output:,}t, 费用≈¥{cost:.4f}")
        return f"Token 使用汇总:\n" + "\n".join(details) + f"\n  总计费用≈¥{total_cost:.4f}"


def encode_image(path: str) -> str:
    with open(path, "rb") as f:
        return base64.b64encode(f.read()).decode("utf-8")

def extract_slides(pptx_path: str) -> list[dict]:
    """解析 PPT：提取文本 + 页面渲染为 PNG"""
    try:
        prs = Presentation(pptx_path)
    except Exception as e:
        log.error(f"无法打开 PPT 文件 {pptx_path}: {e}")
        return []
    
    slides = []
    for i, slide in enumerate(prs.slides, 1):
        title = ""
        body = ""
        notes = ""

        if slide.shapes.title and slide.shapes.title.text:
            title = slide.shapes.title.text.strip()

        for shape in slide.shapes:
            if shape.has_text_frame:
                text = shape.text_frame.text.strip()
                if shape != slide.shapes.title and text:
                    body += text + "\n"

        if slide.has_notes_slide:
            notes = slide.notes_slide.notes_text_frame.text.strip()

        slides.append({
            "index": i,
            "title": title,
            "body": body.strip()[:3000],     # 截断至3000字符（Token 控制，仅影响输入成本）
            "notes": notes[:500]              # 截断至500字符
        })
    
    log.info(f"Phase 1: 提取到 {len(slides)} 页文本")
    return slides


@retry(max_retries=2)
def render_slides_to_images(pptx_path: str, output_dir: str) -> list[str]:
    """将 PPT 每页渲染为 PNG 图片（LibreOffice → PDF → PNG）"""
    os.makedirs(output_dir, exist_ok=True)
    pdf_path = os.path.join(output_dir, "temp.pdf")
    
    try:
        # Step 1: PPT → PDF
        result = subprocess.run(
            ["libreoffice", "--headless", "--convert-to", "pdf",
             pptx_path, "--outdir", output_dir],
            capture_output=True, text=True, timeout=120
        )
        if result.returncode != 0:
            raise RuntimeError(f"LibreOffice 转换失败: {result.stderr}")
        
        # Step 2: PDF → PNG
        images = convert_from_path(pdf_path, dpi=150)
        png_paths = []
        for i, img in enumerate(images, 1):
            path = os.path.join(output_dir, f"page_{i:03d}.png")
            img.save(path, "PNG")
            png_paths.append(path)
        
        return png_paths
    except Exception as e:
        log.error(f"渲染失败: {e}")
        raise
    finally:
        # 无论成功还是失败，都清理临时 PDF
        if os.path.exists(pdf_path):
            os.remove(pdf_path)

@retry(max_retries=3, delay=3)
def llm_extract_meaning_single(slide: dict, image_path: str) -> Optional[dict]:
    """调用本地 Ollama 视觉模型分析单页"""
    
    # 检查图片是否存在
    if not os.path.exists(image_path):
        log.warning(f"图片不存在: {image_path}")
        return None
    
    # 构建 Ollama 多模态请求（/api/chat 格式，qwen-vl 使用 messages 数组）
    prompt = f"""你正在分析一个 PPT 页面的截图及其提取的文本，请严格按 JSON 格式返回：

1. 页面角色：cover/toc/transition/argument/evidence/conclusion/appendix
2. 含义摘要：用 1-3 句话概括这一页在"说什么"
3. 视觉描述：简述页面的视觉结构
4. 语义标签：3-8 个标签

文本提取结果（可能为空）：
标题：{slide['title']}
正文：{slide['body'][:2000]}
备注：{slide['notes'][:500]}

返回格式 JSON：
{{"role": "...", "summary": "...", "visual_desc": "...", "tags": [...]}}"""

    content_parts = [{"type": "text", "text": prompt}]
    content_parts.append({
        "type": "image_url",
        "image_url": {"url": f"data:image/png;base64,{encode_image(image_path)}"}
    })

    payload = {
        "model": VISION_MODEL,
        "messages": [{"role": "user", "content": content_parts}],
        "stream": False,
        "options": {"temperature": 0}
    }

    resp = requests.post(
        f"{OLLAMA_BASE}/api/chat",
        json=payload,
        timeout=120
    )
    resp.raise_for_status()
    result = resp.json()
    
    # 估算 Token
    input_tokens = TokenCounter.estimate(prompt)
    output_text = result.get("response", "")
    output_tokens = TokenCounter.estimate(output_text)
    TokenCounter.add_usage(VISION_MODEL, input_tokens, output_tokens)
    
    # 解析 JSON（从 LLM 回复中提取 JSON 块）
    try:
        # 尝试直接解析
        meaning = json.loads(output_text)
    except json.JSONDecodeError:
        # 尝试从文本中提取 JSON 块
        import re
        json_match = re.search(r'\{[^{}]*\}', output_text, re.DOTALL)
        if json_match:
            meaning = json.loads(json_match.group())
        else:
            log.error(f"无法解析 LLM 返回: {output_text[:200]}")
            return None
    
    log.info(f"  Slide {slide['index']}: semantic_role={meaning.get('role','?')}, "
             f"tags={meaning.get('tags', [])[:3]}...")
    return meaning
```
