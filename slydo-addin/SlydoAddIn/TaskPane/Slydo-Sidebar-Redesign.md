# Slydo 侧边栏重设计 — UI 原型与效果图

> 针对 VSTO 插件端 `SlideRecommendationPane` 的视觉重构
> 设计定稿日期：2026-05-07

---

## 设计理念

**Slydo 品牌调性：** 科技感 · 专业 · 轻量 · 商务 PPT 场景

采用 **亮色商务风格 + 紫色品牌点缀**，整体以白色为底、灰边划分模块、浅蓝悬停反馈，在 Office 的白色界面中保持一致性，同时通过紫色品牌色条（顶部渐变 + 交互按钮）凸显工具的专业感。

## 四、新增功能模块

### 4.1 ~~场景切换 Tab~~ 移除

原三个 Tab（推荐/搜索/大纲）已移除，改为一体式布局：
- 搜索栏始终可见（顶部固定）
- 大纲面板在空白页时自动弹出（插入式）
- 推荐卡片列表填充剩余空间（可滚动）

### 4.2 大纲走向卡片化

当前的大纲走向用横向排列的白色胶囊按钮显示，悬停时紫色边框高亮，每个走向显示方向名 + 页数：

```
┌────────────────────┐ ┌────────────────┐ ┌────────────────┐
│ 应用场景与案例      │ │ 实施效果与价值  │ │ 竞争对比与优势  │
│ 12 页              │ │ 8 页            │ │ 6 页            │
└────────────────────┘ └────────────────┘ └────────────────┘
```

### 4.3 金牌标注视觉增强

金牌 ⭐ 展示在缩略图右上角，20×20px 金色圆形渐变徽章，悬停卡片时徽章微放大（scale 1.15）。

评级规则：
- `LlmScore >= 8` → ⭐ 金色（Gold）
- `LlmScore >= 5` → 🥈 银色（Silver）
- `LlmScore < 5`  → 🥉 灰色（Gray）

### 4.4 加载骨架屏

加载时在卡片区域显示浅灰占位图（"加载中…"），替代原来纯文字提示。

## 五、悬浮预览浮窗（Hover Preview）

### 5.1 交互逻辑

| 触发 | 行为 |
|:---|:---|
| 鼠标悬停卡片 | 延迟 0ms 预热，立即弹出预览浮窗 |
| 鼠标移卡片进浮窗 | 浮窗不消失（可阅读完整信息） |
| 鼠标移出浮窗 | 延迟 200ms 后关闭 |
| 点击预览区域外部 | 浮窗立即关闭（`Deactivate` 事件） |

### 5.2 浮窗结构

```
┌──────────────────┐
│                  │
│   缩略图 270×152  │ ← 16:9 大图，同后端缩略图 API（`/api/v1/thumbnails/{id}`）
│   (16:9)         │
│                  │
├──────────────────┤
│ 企业安全态势感知   │ ← 标题（14px Bold，#333）
│ 基于AI驱动的...   │ ← 摘要（11px Regular，#666，最多4行）
│                  │
│ 85% 匹配  ⭐金牌  │ ← 匹配度（#009900/#c89600/#c83232 按阈值）、标签（浅灰或金色背景）
│ 联软安全方案     │ ← 来源（10px，#aaa）
└──────────────────┘
  ▶ (箭头指向侧边栏)
```

### 5.3 定位逻辑

浮窗显示在**侧边栏左侧**（覆盖 PPT 编辑区域），与侧边栏顶部对齐（Y轴偏移 +10px 避免遮挡顶部渐变条）。

- `X` = 侧边栏左侧坐标 - 浮窗宽度 - 4px
- `Y` = 侧边栏顶部坐标 + 10px
- 浮窗右侧有一个小三角箭头指向侧边栏（Y=30px 位置）

### 5.4 缩略图尺寸对比

| | 卡片内缩略图 | 预览浮窗缩略图 |
|:---|:---:|:---:|
| 宽度 | **130px** | **270px** |
| 高度 | **74px** | **152px** |
| 比例 | 16:9 | 16:9 |
| 信息量 | 标题 + 摘要2行 | 标题 + 摘要4行 + 标签 + 来源 |

## 六、代码实现

### 6.1 技术选型：WPF（通过 ElementHost 托管）

> ⚠️ **选型说明：** 最初用 WinForms 实现，但 WinForms 的渲染能力（无圆角、无渐变、无阴影、无动画）
> 与 HTML 效果图差距过大，最终改用 WPF。VSTO 插件通过 `ElementHost` 托管 WPF UserControl。

**改用 WPF 的步骤：**

1. 创建 WPF UserControl（`SlideCardWpf` + `SlidePreviewWpf` + `SlideRecommendationWpf`）
2. 在 `.csproj` 中添加 WPF 引用（System.Xaml、WindowsBase、PresentationCore、PresentationFramework）
3. 添加 `<Page Include="...xaml">` 引用
4. 修改 `SlideRecommendationPane.cs` 用 `ElementHost` 托管 WPF 控件

### 6.2 文件清单

| 文件 | 类型 | 说明 |
|:---|:---|:---|
| `SlideCardWpf.xaml` + `.xaml.cs` | 新建 | WPF 卡片控件，圆角+阴影+动画+金牌+导入按钮 |
| `SlidePreviewWpf.xaml` + `.xaml.cs` | 新建 | WPF 预览浮窗 Window，圆角白底+阴影+三角箭头 |
| `SlideRecommendationWpf.xaml` + `.xaml.cs` | 新建 | WPF 侧边栏主控件（完整布局+逻辑+预览集成） |
| `SlideRecommendationPane.cs` | 修改 | WinForms 容器层，用 ElementHost 托管 WPF |
| `sidebar-mockup.html` | 效果图 | HTML 高保真原型 |
| `Slydo-Sidebar-Redesign.md` | 本文件 | 设计文档 |

### 6.2 关键 API

| API | 用途 | 现有 |
|:---|:---|:---:|
| `GET /api/v1/thumbnails/{slide_id}` | 缩略图获取 | ✅ 已有 |
| `GET /api/v1/recommend/export?slide_id=...` | 单页导出 | ✅ 已有 |
| `GET /api/v1/recommend/slides?q=...` | 推荐/搜索 | ✅ 已有 |

### 6.3 编译说明

- 项目：VSTO 插件，`.NET Framework 4.8+`
- 在 Visual Studio 2022+ 中编译
- 需要引用：`System.Windows.Forms`、`System.Drawing`
- `SlidePreviewForm.cs` 需添加到项目文件中（`SlydoAddIn.csproj`）

## 七、后续优化方向
